## 1. 产品概述

### 1.1 产品定位

MiraPrep 是一款基于大语言模型的仿真模拟面试产品。用户上传简历后，AI 面试官基于简历内容、目标岗位和用户补充信息，进行一轮贴近真实场景的完整面试（支持语音与文字交流），并在面试结束后输出评级与逐题分析报告，帮助求职者发现短板、迭代表现。

### 1.2 核心价值

- **仿真性**：面试官具备真实面试的节奏——开场寒暄、简历深挖、技术/业务考察、追问、反问环节、收尾，而非机械问答。
- **个性化**：问题从用户简历和目标岗位生成，追问基于用户上一轮回答动态展开。
- **可复盘**：每次面试沉淀为结构化报告（逐题回答、参考答案、耗时、改进建议），形成成长记录。

### 1.3 目标用户

| 用户画像 | 场景 |
|---|---|
| 应届生/实习生 | 缺乏面试经验，需要低成本高频次演练 |
| 社招跳槽者 | 针对特定目标公司/岗位做冲刺准备 |
| 转行者 | 检验新技术栈掌握程度，暴露知识盲区 |

### 1.4 MVP 范围

本文档描述的全部页面与流程均属 v1.0 范围；多语言、面试官人设商店、团队版、模拟群面等列入后续规划（见第 7 章 Backlog）。

## 3. 页面功能需求

### 3.1 落地页（Landing Page）

**目标**：3 秒内让访客理解产品是什么，引导注册转化。

**页面结构（自上而下）**

1. **导航栏**：左侧 Logo + 产品名；中部锚点导航（功能 / 工作原理 / 常见问题）；右侧「登录」按钮（ghost 样式）+「免费开始」主按钮。滚动时导航栏变为半透明毛玻璃背景（backdrop-blur）并出现细底边。
2. **Hero 区**：
   - 主标题（一句话价值主张，如「你的下一场面试，先在这里赢一次」）+ 副标题。
   - 主 CTA「免费开始模拟面试」+ 次 CTA「观看演示」。
   - 视觉主体：面试对话界面的动态模拟演示（自动播放的打字动画，展示 AI 面试官提问 → 用户回答 → 追问的过程），而非静态截图。
3. **信任/数据条**：滚动数字动画展示「累计面试场次 / 覆盖岗位数 / 平均评级提升」。
4. **功能特性区**：Bento Grid 布局（不等宽卡片网格），展示 4–6 个核心能力：简历深挖提问、语音实时对话、动态追问、逐题报告、评级体系、多岗位题库。卡片 hover 时轻微上浮 + 边框高亮。
5. **工作原理区**：3 步横向流程（上传简历 → 仿真面试 → 获得报告），滚动进入视口时依次淡入。
6. **报告样例展示区**：以设备框（浏览器 mock 窗口）展示一份脱敏的面试报告截图/交互 demo。
7. **FAQ 手风琴** + **底部 CTA 区** + **Footer**（产品链接、条款、隐私政策）。

**交互要求**

- 全页滚动驱动动画（scroll-triggered fade/slide-in），使用 IntersectionObserver 或 Framer Motion 的 `whileInView`，动画时长 ≤ 400ms，不阻塞阅读。
- 已登录用户访问 `/` 时，导航栏「登录」替换为「进入工作台」。

### 3.2 登录/注册页（`/auth`）

**布局**：左右分栏。左侧为品牌区（深色背景 + 产品插画/动态渐变 + 一句 slogan + 用户评价轮播），右侧为表单区（居中卡片，最大宽度 400px）。移动端隐藏左侧品牌区。

**功能需求**

- 登录与注册通过顶部 Tab 或底部文字链切换，切换时表单区域平滑过渡（高度动画 + 淡入淡出）。
- **注册**：邮箱 + 密码 + 确认密码；密码强度实时指示条；邮箱格式实时校验（失焦触发）；发送邮箱验证码（60s 倒计时重发）。
- **登录**：邮箱 + 密码；「记住我」；「忘记密码」走邮箱重置流程。
- **第三方登录**（v1.0 至少一种）：GitHub / Google OAuth，按钮置于表单上方，以分割线「或」与邮箱表单分隔。
- 错误反馈：表单内联错误文案（红色小字 + 输入框红边），全局错误（如密码错误）用输入框轻微 shake 动画 + toast。
- 登录成功：按钮进入 loading 态（spinner 替换文字）→ 跳转 `/dashboard`；若 `isFirstLogin=true`，工作台加载后自动弹出引导配置。

### 3.3 初次登录引导配置（Onboarding Modal）

**形态**：全屏遮罩 + 居中大尺寸模态（桌面约 640px 宽），分 3 步，顶部有步骤进度指示（分段进度条），支持「跳过」（右上角，跳过后可在设置中补填，工作台会显示补填提醒卡片）。

| 步骤 | 内容 | 控件 |
|---|---|---|
| ① 目标岗位 | 岗位方向（如后端/前端/算法/产品/测试/数据…） | 大号可选卡片网格（带图标），单选后自动进入下一步 |
| ② 技术栈与经验 | 技术栈多选（随岗位方向联动的标签云，支持搜索补充）；工作年限（应届/1-3/3-5/5-10/10+）；当前状态（在职/离职/学生） | 标签多选 + 分段选择器 |
| ③ 求职意向 | 期望公司类型（大厂/外企/创业/国企）；期望城市；求职紧迫度 | 可选卡片 + 下拉 |

**交互**：步骤切换为横向滑动过渡；完成后播放一个轻量的完成动画（对勾 + 彩带微粒），文案「配置完成，去开始你的第一场面试吧」，CTA 直达面试配置向导。

### 3.4 个人工作台（`/dashboard`）

**布局**：左侧固定侧边栏（Logo、导航：工作台 / 面试记录 / 简历库 / 设置；底部为用户头像菜单）+ 右侧内容区。侧边栏可折叠为图标模式。

**内容区模块**

1. **欢迎横幅**：问候语（随时间变化：早上好/下午好）+ 一句激励文案 + 主按钮「开始新面试」（页面视觉焦点，尺寸大、渐变/高对比配色）。
2. **数据概览卡片行**（4 张统计卡）：累计面试次数、最近评级、平均得分趋势（迷你 sparkline）、累计练习时长。数字进入视口时滚动动画。
3. **能力雷达图**：基于历史面试的五维能力评估（专业知识、项目深度、表达逻辑、临场应变、岗位匹配度），随面试次数增加而逐渐精准；不足 1 次面试时显示占位空态。
4. **面试记录列表**：卡片式列表，每条含：岗位名称、日期、时长、评级徽章（S/A/B/C/D 不同配色）、状态（已完成/中途退出）。点击进入对应报告页。支持按岗位/评级筛选、按时间排序。空态：插画 +「还没有面试记录，开始第一场吧」+ CTA。
5. **简历库入口**：显示已上传简历数量与最近使用的简历；管理页支持上传、重命名、删除、预览、设为默认。

### 3.5 面试配置向导（`/interview/setup`）

**形态**：独立页面，顶部横向 Stepper（① 简历 → ② 岗位 → ③ 补充信息），下方为当前步内容，底部固定操作栏（上一步/下一步）。步骤间切换保留已填内容；刷新页面通过本地草稿恢复。

**Step 1 — 选择/上传简历**

- 两个 Tab：「我的简历库」（已上传简历卡片单选，含文件名、上传时间、页数）与「上传新简历」。
- 上传：拖拽区（dashed 边框，拖入时高亮）+ 点击选择。支持 PDF / DOCX，≤ 10MB。
- 上传后进入解析流程：进度条 → 解析成功后展示**简历解析预览卡**（提取出的姓名、教育经历、工作经历、项目、技能标签），用户可确认或手动修正关键字段。解析失败给出明确原因与重试。

**Step 2 — 岗位与难度**

- 岗位方向 + 具体岗位（如「后端开发 — Java」），默认预填 Onboarding 中配置的目标岗位。
- 支持粘贴**目标岗位 JD**（文本域），Agent 将结合 JD 出题。
- 面试难度：校招 / 社招初级 / 社招资深 / 专家级。
- 面试类型多选：技术一面（基础+项目）/ 技术二面（深挖+系统设计）/ HR 面 / 综合面。
- 面试时长：15 / 30 / 45 分钟。

**Step 3 — 补充要求**

- 自由文本域：「告诉面试官你想重点练习什么」（如：重点考察分布式、少问算法、帮我练自我介绍…）。
- 面试官风格选择（可选卡片）：温和引导型 / 标准专业型 / 高压追问型。
- 语音开关预设：默认开启语音模式或纯文字模式。

**确认页（Step 3 底部）**：汇总卡片展示全部配置 →「开始面试」按钮。点击后过场动画（「面试官正在阅读你的简历…」进度提示，此时后端完成简历分析与题目规划）→ 进入面试页。

### 3.6 面试进行页（`/interview/[sessionId]`）

**整体氛围**：沉浸式，隐藏全局侧边栏，深色或极简浅色主题，减少干扰。

**布局（桌面）**

- **顶部状态栏**：岗位名称、面试阶段指示（自我介绍 → 简历深挖 → 专业考察 → 反问 → 收尾，当前阶段高亮）、计时器、「结束面试」按钮（二次确认弹窗）。
- **左/中区 — 面试官区**：AI 面试官形象（抽象声波球体/头像，说话时有呼吸/波动动画），面试官当前问题以流式打字方式呈现，同时 TTS 播报（可静音）。
- **主对话流**：类聊天界面，面试官消息居左、用户回答居右；面试官提问支持代码块渲染（考代码题时）。自动滚动到最新消息，用户上滑查看历史时暂停自动滚动并显示「回到最新」浮动按钮。
- **底部输入区**：
  - **文字模式**：多行输入框（Enter 发送 / Shift+Enter 换行），支持 Markdown 代码块输入。
  - **语音模式**：大号麦克风按钮，按住说话或点击开始/结束；录音时显示实时声纹波形动画 + 实时 ASR 转写文字（可在发送前编辑修正）；静音超过 N 秒提示「是否结束本次回答」。
  - 两种模式随时切换（输入区右侧切换按钮），一次面试中可混用。
  - 辅助操作：「请重复一下问题」「跳过这题」（跳过将计入报告）。

**关键交互与状态**

- 面试官回复流式输出（SSE/WebSocket），伴随「正在思考」的三点动画占位。
- 弱网/断线：本地提示 + 自动重连，会话状态由后端持久化，刷新页面可恢复到当前进度。
- 中途退出：确认弹窗（「退出后本场面试将标记为未完成，已回答部分仍会生成部分报告」）。
- 防误触：面试期间关闭浏览器 tab 触发 `beforeunload` 提醒。
- 无障碍：语音播报可关、字号可调、全键盘可操作。

**面试结束触发**：① 题目流程走完由 Agent 主动收尾；② 时长到达上限时 Agent 自然收尾；③ 用户手动结束。结束后进入评分生成过场页（进度动画 + 阶段文案「正在分析你的回答…」「正在生成参考答案…」），完成后跳转评级结果页。

### 3.7 评级结果页（`/interview/[sessionId]/result`）

**目标**：制造「成绩揭晓」的仪式感。

- 居中大号评级徽章（S / A / B / C / D），揭晓时有翻转/缩放 + 微粒动画；不同评级不同配色（S 金、A 绿、B 蓝、C 橙、D 灰红）。
- 评级下方：总分（如 82/100）、一句话总评、本场关键数据（时长、答题数、平均回答时长）。
- 五维雷达图（与工作台同一模型），与用户历史平均值叠加对比。
- 亮点与不足各 Top3（简短 bullet）。
- 操作：主按钮「查看完整报告」→ `/report/[sessionId]`；次按钮「返回工作台」「再来一场」。

### 3.8 面试报告页（`/report/[sessionId]`）

**布局**：顶部报告摘要区（评级徽章、总分、岗位、日期、配置回显、雷达图）+ 左侧题目目录锚点导航（桌面端 sticky）+ 右侧逐题详情主体。

**逐题详情卡片**（每题一张，默认收起为摘要行，点击展开）

| 字段 | 说明 |
|---|---|
| 题目 | 面试官原始问题 + 所属阶段/考察点标签 |
| 我的回答 | 完整回答文本（语音回答显示转写文本，可回放原音频） |
| 单题得分 | 0–10 分 + 颜色条 |
| 耗时 | 思考时长 + 回答时长，与「建议时长」对比（超时标橙） |
| 参考答案 | 结构化参考答案（要点式），可展开「深入阅读」查看完整版 |
| 追问链 | 该题下的追问与回答，以缩进时间线呈现 |
| 改进建议 | 2–4 条针对性建议（内容、结构、表达三个维度） |

**其他功能**

- 「重练此题」：单题进入迷你练习模式（后续版本可先做占位入口）。
- 报告导出 PDF；生成分享链接（默认关闭，开启后脱敏处理姓名等信息）。
- 历史对比：同岗位多次面试的得分折线。

## 4. 视觉与交互设计规范

### 4.1 设计基调

现代、扁平、克制的高级感。参考基准：**Linear**（排版节奏、深色模式、微动效）、**Vercel**（极简黑白灰 + 单一强调色、清晰层级）、**Stripe**（落地页叙事、渐变运用）、**Raycast**（玻璃质感、键盘友好）。原则：大量留白、低饱和底色 + 单一高饱和强调色、层级靠字重与间距而非线框、动效轻且快。

### 4.2 色彩系统（Tailwind 语义化 token）

| Token | 用途 | 浅色模式 | 深色模式 |
|---|---|---|---|
| `background` | 页面底色 | `#FAFAFA` | `#0A0A0B` |
| `surface` | 卡片/面板 | `#FFFFFF` | `#141416` |
| `border` | 描边分隔 | `#E5E7EB` | `#26262A` |
| `foreground` | 主文本 | `#111113` | `#F4F4F5` |
| `muted` | 次文本 | `#6B7280` | `#9CA3AF` |
| `primary` | 品牌强调色 | 靛蓝-紫渐变主色 `#6366F1`（可微调品牌色） | 同左，深色下略提亮 |
| `success / warning / danger` | 语义色 | `#10B981` / `#F59E0B` / `#EF4444` | 同系提亮 |

评级配色：S `#EAB308`（金）、A `#10B981`、B `#3B82F6`、C `#F59E0B`、D `#EF4444`。

- 全站支持**深色/浅色模式**（`next-themes`，跟随系统 + 手动切换）；面试进行页默认深色。
- 渐变仅用于：落地页 Hero、主 CTA、评级徽章；正文界面保持纯色扁平。

### 4.3 字体与排版

- 中文：`system-ui` 栈优先（PingFang SC / Microsoft YaHei），西文与数字：`Inter`；代码/计时器/得分：`JetBrains Mono`。
- 字号阶梯：display 48–64（落地页）、h1 30、h2 24、h3 18、正文 15–16、辅助 13、标签 12。行高正文 1.6、标题 1.2。
- 数字型数据（得分、计时）使用 `tabular-nums` 避免跳动。

### 4.4 布局与间距

- 间距使用 4px 基数网格；卡片内边距 24px；区块间距 64–120px（落地页）/ 24–32px（应用内）。
- 圆角：卡片 12–16px、按钮 8–10px、输入框 10px、徽章 full。
- 阴影极轻（`shadow-sm`），层级主要靠 `border` + 背景色差；弹层用 `shadow-lg` + 遮罩 `black/40 + backdrop-blur-sm`。
- 内容最大宽度：落地页 1200px、应用内 1440px、报告正文 880px。

### 4.5 动效规范

- 库：Framer Motion；时长：微交互 150–200ms、页面过渡 250–350ms、仪式性动画（评级揭晓）600–900ms；缓动 `ease-out` / `spring(stiffness 300, damping 30)`。
- 必备动效清单：按钮 hover 亮度/位移、卡片 hover 上浮 2px、页面切换淡入+上移 8px、Stepper 步骤滑动、流式打字光标、声纹波形、评级揭晓、数字滚动。
- 遵循 `prefers-reduced-motion`：用户开启减弱动效时全部动画降级为淡入淡出。

### 4.6 组件库与实现约定

- 基础组件：**shadcn/ui**（Button/Input/Dialog/Tabs/Select/Toast/Skeleton…）+ Tailwind；图表：Recharts（雷达图、折线、sparkline）；图标：lucide-react。
- 所有异步内容使用 **Skeleton 占位**（工作台统计卡、记录列表、报告页），禁止布局跳动（CLS）。
- 统一空态组件：插画 + 标题 + 描述 + CTA。
- Toast 统一右上角，自动消失 4s，错误类可手动关闭。
- 响应式断点：`sm 640 / md 768 / lg 1024 / xl 1280`；核心流程（配置、面试、报告）需完整适配移动端，面试页移动端输入区改为底部全宽、语音按钮加大。

## 5. AI 面试官 Agent 设计（FastAPI 侧）

### 5.1 面试流程状态机

```
GREETING（开场寒暄+流程说明）
   ▼
SELF_INTRO（自我介绍，1 题）
   ▼
RESUME_DEEP_DIVE（简历/项目深挖，2–4 题，每题 0–3 层追问）
   ▼
DOMAIN_ASSESSMENT（专业能力考察：基础知识/场景设计/编码思路，随难度与面试类型配置，3–6 题）
   ▼
BEHAVIORAL（行为面/HR 问题，0–2 题，HR 面类型时为主体）
   ▼
CANDIDATE_QA（反问环节：邀请用户提问并作答）
   ▼
CLOSING（收尾致谢，说明报告生成）
```

- 各阶段题量随「面试时长」配置动态伸缩；计时到点时状态机强制推进到 CANDIDATE_QA。
- **题目规划**：面试开始前（配置确认后的过场阶段），Agent 基于简历解析结构 + 岗位 + JD + 补充要求生成「面试大纲」（题目池 + 考察点映射），面试中按大纲执行并允许动态偏移。

### 5.2 追问与对话策略

- 每轮回答后 Agent 做即时评估（完整性/深度/真实性信号），据此决定：追问（回答有亮点或疑点）、给提示（明显卡壳）、换题（多次无效回答）。
- 追问深度上限 3 层，避免死磕；「高压追问型」风格提高追问概率与尖锐度，「温和引导型」增加提示与鼓励。
- Agent 需处理的边界情况：用户答非所问（礼貌拉回）、用户反问问题本身（澄清但不给答案）、用户长时间沉默（语音模式下主动询问）、不当内容（专业地终止话题）。
- 面试全程 Agent **不当场评价对错、不打分**，保持真实面试官口吻；所有评价延迟到报告。

### 5.3 评分与评级体系

- **单题评分**（0–10）：由评估模型在面试结束后统一批改，维度权重随题型变化（知识准确性、深度、结构化表达、结合自身经验）。
- **五维能力分**（0–100）：专业知识、项目深度、表达逻辑、临场应变、岗位匹配度，由逐题得分按考察点映射聚合。
- **总分** = 五维加权（权重随面试类型配置）；**评级映射**：S ≥ 90，A 80–89，B 70–79，C 60–69，D < 60。
- **参考答案生成**：结合题目考察点 + 用户简历背景生成（参考答案应贴合用户自身项目，而非通用答案）。
- 未完成面试：已答部分正常批改，报告标注「部分完成」，不计入评级统计趋势。

### 5.4 语音链路

- **ASR**：前端采集音频（MediaRecorder，16kHz）→ WebSocket 流式推送 FastAPI → 流式转写实时回显（可接入 Whisper 或云厂商流式 ASR）。
- **TTS**：面试官文本流式合成播报，句级切分以降低首音延迟；目标首字延迟 < 1.5s。
- 转写文本发送前允许用户编辑；原始音频保留用于报告回放（可在设置中关闭存储）。

## 6. 系统架构与接口设计

### 6.1 总体架构

```
Next.js 前端 ──REST──► Spring Boot 业务服务 ──► MySQL / Redis / 对象存储(OSS/S3)
     │                        │
     │                        └─内部调用（REST/gRPC）─► FastAPI AI 服务
     │                                                    │
     └────WebSocket/SSE（面试会话、语音流）────────────────┘
                                              FastAPI ──► LLM API / ASR / TTS / 向量库
```

**职责划分**

| 服务 | 职责 |
|---|---|
| Spring Boot | 用户认证（JWT + Refresh Token）、用户资料与偏好、简历文件管理（上传至对象存储）、面试会话生命周期与记录、报告持久化与查询、统计聚合 |
| FastAPI | 简历解析（PDF/DOCX → 结构化 JSON）、面试大纲生成、面试官对话推理（流式）、ASR/TTS 网关、批改与报告内容生成 |

### 6.2 核心数据模型

```
User(id, email, password_hash, avatar, created_at, is_first_login)
UserProfile(user_id, job_direction, tech_stacks[], experience_level, status, preferences...)
Resume(id, user_id, file_url, file_name, parsed_json, is_default, created_at)
InterviewSession(id, user_id, resume_id, job_direction, job_title, jd_text,
                 difficulty, types[], duration_min, custom_requirements,
                 interviewer_style, status[created|ongoing|completed|aborted],
                 started_at, ended_at)
InterviewMessage(id, session_id, role[interviewer|candidate], content,
                 audio_url?, phase, question_id?, created_at)
Question(id, session_id, phase, text, focus_points[], order,
         think_seconds, answer_seconds, suggested_seconds, skipped)
Report(id, session_id, grade, total_score, dimension_scores{}, summary,
       highlights[], weaknesses[], created_at)
QuestionReview(id, report_id, question_id, score, reference_answer,
               suggestions[], follow_up_chain_json)
```

### 6.3 API 设计（Spring Boot，`/api/v1`）

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/auth/register` `/auth/login` `/auth/refresh` `/auth/send-code` | 认证 |
| GET/PUT | `/users/me` `/users/me/profile` | 资料与 Onboarding 配置 |
| POST | `/resumes`（multipart 上传，触发 FastAPI 解析） | 简历上传 |
| GET/DELETE/PATCH | `/resumes` `/resumes/{id}` | 简历库管理 |
| POST | `/interviews`（body=配置向导结果，返回 sessionId，异步生成大纲） | 创建面试 |
| GET | `/interviews/{id}/status` | 大纲准备状态（前端过场页轮询） |
| POST | `/interviews/{id}/end` | 手动结束 |
| GET | `/interviews?page&filter` | 面试记录列表 |
| GET | `/reports/{sessionId}` | 报告详情 |
| GET | `/reports/{sessionId}/export` | PDF 导出 |
| GET | `/stats/overview` | 工作台统计与雷达数据 |

### 6.4 实时接口（FastAPI）

| 协议 | 路径 | 说明 |
|---|---|---|
| WebSocket | `/ws/interview/{sessionId}` | 双向：用户文本/音频帧 ↑，面试官流式文本 token、TTS 音频帧、阶段变更事件、ASR 部分转写 ↓ |
| SSE（备选） | `/interviews/{id}/stream` | 纯文字模式下可用 SSE 替代 |
| POST | `/internal/resumes/parse` | Spring Boot 内部调用，简历解析 |
| POST | `/internal/interviews/{id}/grade` | 面试结束触发批改，完成后回调 Spring Boot 落库 |

消息协议采用 JSON envelope：`{type: "token"|"audio"|"asr_partial"|"phase_change"|"interview_end"|"error", payload, seq}`；断线重连以 `seq` 续传。

### 6.5 非功能需求

- **性能**：落地页 LCP < 2s；面试官首 token < 2s；ASR 转写延迟 < 800ms；报告生成 < 60s（过场页展示进度）。
- **安全与隐私**：简历/音频私有存储（签名 URL）；密码 bcrypt；接口限流；LLM 输入防注入（简历内容作为数据而非指令处理）；提供「删除我的全部数据」入口。
- **可靠性**：面试会话状态持久化（Redis + DB），前端断线重连恢复；批改任务队列化（失败重试）。
- **可观测**：会话级 trace（每场面试的 LLM 调用链）、Token 成本统计。
