# T-116 · 面试官动画 + 评级揭晓动效 + 雷达 + 面试页深色

| 轨道 | 里程碑 | 预估 | 依赖 | 阻塞 |
|---|---|---|---|---|
| Frontend | M2 | 1.5d | T-108, T-006 | — |

## 背景
补齐仿真感与仪式感的视觉细节。先读 PRD §3.6/§3.7、§4.5 动效规范。

## 目标
面试官声波球「说话呼吸」动画、流式打字光标、评级揭晓仪式动效（翻转/缩放 + 微粒）、五维雷达打磨、面试进行页默认深色主题。

## 范围
- **做**：面试官形象说话时动画（结合 TTS 播放状态）、流式打字光标、评级页揭晓动画（评级徽章翻转/缩放 + 微粒 + 数字滚动）、雷达图视觉打磨（与历史对比叠加）、面试进行页深色主题（PRD §4.2「面试页默认深色」）。
- **不做**：不改数据链路（T-104/T-108/T-114 已接）；通用动效体系在 T-115。

## 技术规格
- 用 Framer Motion（若 T-115 已引入则复用其规范；否则本任务可先局部引入并与 T-115 对齐）。
- 时长遵循 PRD §4.5：微交互 150–200ms、仪式动画 600–900ms；`spring(stiffness 300, damping 30)`。
- 评级揭晓：徽章 scale/flip + 粒子；总分数字滚动（`tabular-nums`）。
- 面试页深色：用 T-006 的 token 深色变体；保证与 View Transitions、语音波形共存。
- 遵守 `prefers-reduced-motion`（降级为淡入淡出）。

## 涉及文件
- 修改 `src/components/interview/InterviewClient.tsx`（说话动画、深色、打字光标）
- 修改 `src/app/interview/[sessionId]/result/page.tsx`（揭晓动效、数字滚动）
- 修改 `src/components/report/RadarChart.tsx`（打磨）

## 验收标准
1. 面试官说话时有呼吸/波动动画，与 TTS 状态同步。
2. 评级页有揭晓仪式动效 + 数字滚动，不同评级不同配色。
3. 面试进行页深色主题完整可用、对比度良好。
4. 雷达图视觉精致、历史对比清晰。
5. reduced-motion 下全部降级；`lint`/`build` 通过。

## 验证方式
PR 贴：面试页深色截图、评级揭晓录屏、雷达图截图、reduced-motion 对比。
