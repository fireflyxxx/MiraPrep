# T-110 · 安全加固（限流 + 防注入 + 数据删除）

| 轨道 | 里程碑 | 预估 | 依赖 | 阻塞 |
|---|---|---|---|---|
| Full-stack | 横切 | 1.5d | T-010, T-040 | — |

## 背景
落地 PRD §6.5 的安全与隐私要求，作为上线前的收口。先读 `DEVELOPMENT.md §7.5`。

## 目标
统一限流、LLM 防注入复核、私有存储签名、「删除我的全部数据」入口。

## 范围
- **做**：接口限流（登录/发码/上传/面试创建）、签名 URL 复核（简历/音频私有）、LLM 输入防注入统一复核（简历/回答作数据）、密码策略、「删除我的全部数据」端到端（账号/简历/会话/报告/对象存储/音频）、基础安全响应头/CORS 收紧。
- **不做**：不做渗透测试级别加固；不引入 WAF 等基础设施。

## 技术规格
- 限流：Redis 计数（IP + 用户维度），关键端点（`/auth/*`、`/resumes`、`/interviews`），超限 `code=42900`。
- 存储：所有简历/音频私有，一律签名 URL、短时效；校验归属。
- 防注入：审查 T-021/T-031/T-040/T-105 的 prompt，确保用户/简历内容都在「不可信数据」区、有明确防注入声明；加统一的输入清洗/长度限制。
- 数据删除：`DELETE /users/me`（需二次确认/密码）→ 级联删除该用户账号、简历（含对象存储对象）、会话、消息、报告、音频；不可逆，删除前给导出提示（可选）。
- 安全头：合理 CSP/CORS、`Secure/HttpOnly` cookie（若用 cookie 存 refresh）。

## 涉及文件
- 后端 `web/RateLimitFilter.java` 或拦截器、`user/DataDeletionService.java` + Controller、存储签名复核、Security 头配置
- AI 侧 prompt 防注入复核（各 `app/prompts/*`）
- 前端「删除我的全部数据」入口（设置页；若设置页未建则最小实现）

## 验收标准
1. 关键端点限流生效（超限返回 42900）。
2. 简历/音频仅可通过签名 URL 访问，越权拿不到。
3. prompt 防注入复核通过（针对 T-021/031/040/105 各跑注入样例，不被劫持）。
4. 「删除我的全部数据」能级联删净（DB + 对象存储），不可逆且有确认。
5. 安全响应头/CORS 合理。

## 验证方式
PR 贴：限流触发、越权访问被拒、注入样例防御、数据删除前后 DB/存储对比。
