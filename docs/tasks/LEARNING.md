# 学习模式 · 教学协议与知识地图

> 本项目不只是「把功能做出来」，也是 owner 一边用 AI 开发、一边**从零学全栈**的学习载体。
> 任何 agent 执行 `docs/tasks/` 里的任务时，除了完成任务，**还必须按本文件的「教学协议」边做边教**。

---

## 1. 学习者档案（agent 请按此校准讲解深浅）

- **水平**：编程新手，前端和后端（Java / Python / 数据库）都在从头学。
- **偏好**：**讲解为主**——动手前先讲清思路，动手后做总结。
- **含义**：
  - 不要假设任何前置知识。**第一次出现的术语必须解释**（一句人话 + 尽量打个生活比方）。
  - 少跳步。宁可多说一句「为什么」，也不要甩一堆代码让他自己看懂。
  - 多用类比，少堆术语；术语和英文缩写第一次出现时给中文解释。
  - 主动点出「新手常踩的坑」。

---

## 2. 教学协议（每个任务都要走这套流程）

### 🟢 动手前（讲思路）
1. **放进大图**：用 2–4 句说清「这个任务在整个系统里是干什么的、为什么需要它、和前后哪一步衔接」。可对照 `../DEVELOPMENT.md §2` 的架构图。
2. **讲清核心概念**：把本任务用到的关键概念（见 §4 每个任务的「你会学到」）逐个用「一句话人话 + 一个类比」讲明白。
3. **讲实现思路**：说明打算分几步做、每步为什么这么做，让 owner 心里有地图，而不是看你突然写出一堆文件。

### 🟡 动手中（关键处点一句）
- 写关键代码时，顺带说「这段在干什么、为什么这么写」，重要的行加中文注释。
- 遇到**技术选择**（比如「为什么用 JWT 不用 session」「为什么解析放 FastAPI 不放 Java」），停下来用一两句讲清取舍。
- 避免术语轰炸；新概念即时解释。

### 🔵 动手后（做总结）
用固定小结构收尾（放在任务完成的回复里）：
- **📌 这次学到了什么**：3–5 条要点（人话）。
- **🤔 你现在应该能回答**：2–3 个自测小问题（帮 owner 检验，不强制答）。
- **🔗 想深入可以看**：1–2 个权威链接（优先官方文档，见 §6）。

### 风格要求
- 全程中文、口语化。
- 报喜也报忧：卡住了、跳过了、测试没过，如实说，并解释原因。
- owner 说「这块没懂」时，换个更简单的角度再讲一遍，别重复原话。

## 5. 核心术语速查（新手版一句话解释）

反复会遇到的词，先建立直觉，细节让 agent 在任务里展开：

- **前端 / 后端 / 全栈**：前端=用户在浏览器看到和点的界面；后端=看不见的服务器逻辑与数据；全栈=两者都做。
- **REST API / 端点(endpoint)**：后端对外提供的一个个「功能入口」，前端按约定的网址+方法去调用。一个端点就像餐厅菜单上的一道菜。
- **HTTP 方法 / 状态码**：GET 取数据、POST 新建、PUT 改、DELETE 删；返回码 200 成功、400 你传错了、401 没登录、404 找不到、500 服务器炸了。
- **JSON**：前后端之间传数据的通用格式，就是带括号的键值对文本。
- **CSR / SSR**：页面在浏览器里渲染（客户端）还是在服务器上先渲染好再发过来（服务端）。Next.js 两者都能做。
- **组件 / 状态(state)**：组件=可复用的界面积木；状态=组件当前记着的数据（比如输入框里的文字），状态变了界面就重渲染。
- **数据库 / SQL / 表**：长期存数据的地方；SQL 是操作它的语言；数据像 Excel 一样按「表」组织。
- **主键 / 外键 / 索引**：主键=每行的唯一身份证号；外键=指向另一张表的引用；索引=为了查得快建的「目录」。
- **ORM / 实体**：让你用写对象的方式操作数据库、不用手写 SQL 的工具；实体=和数据库表一一对应的类。
- **迁移(migration)**：用版本化的脚本管理数据库结构变化，像给数据库做 git。
- **认证 vs 授权**：认证=证明你是谁（登录）；授权=你能干什么（权限）。
- **JWT / token**：登录后服务器发给你的一张「电子通行证」，之后每次请求带上它证明身份。
- **哈希 / bcrypt**：把密码变成不可逆的乱码再存，即使库被偷也拿不到原密码。
- **依赖注入**：需要什么零件不自己 new，由框架自动「送」进来，方便替换和测试。
- **环境变量 / .env**：把密钥、数据库地址等**不写死在代码里**、按环境注入的配置。
- **Docker / 容器 / 镜像**：镜像=打包好的软件模板；容器=用镜像跑起来的一个隔离小盒子。让「在我电脑能跑」变成到处都能跑。
- **对象存储 / 签名 URL**：专门存文件（简历/音频）的云仓库；签名 URL=一张有时效的临时下载凭证，防止文件被随便访问。
- **SSE / WebSocket**：让服务器能主动、持续把数据推给前端的两种技术；SSE 单向（适合面试官流式打字），WebSocket 双向（适合语音）。
- **LLM / prompt / 流式**：大语言模型（Claude）；prompt=你给模型的指令与上下文；流式=一个字一个字往外吐，不用等整段生成完。
- **缓存 / Redis**：把频繁用的数据放在内存里读得飞快；Redis 是常用的内存数据库，这里用来存面试会话临时态。
- **异步 / 回调**：不干等结果，先返回、后台慢慢做；做完再「回调」通知对方。简历解析、批改都是这样。
- **提示注入(prompt injection)**：用户或简历里藏「忽略以上指令」之类的话试图劫持 LLM；防御=把用户内容当「数据」而非「指令」处理。
